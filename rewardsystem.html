Reward System

Take an in-depth look at the detailed code implementation of Reward System by visiting this Repository link provided for the Project :)

Description

  • Developed a dynamic reward system in Unity that generates random chests categorized as common, rare, epic, and legendary. Players can unlock these chests using a timer or by spending coins, creating an engaging and rewarding gameplay experience. This system adds an element of chance and strategy, enhancing player interaction and enjoyment.

Features

  • Unlocking: Implemented a queue-based unlocking system where chests set to unlock via a timer are queued sequentially. Once the first chest is unlocked, the timer for the next chest in the queue begins automatically. Players also have the option to bypass the timer and unlock chests instantly using coins, adding flexibility and enhancing the gameplay experience.
  • System UI: Developed a UI notification system that triggers after unlocking a chest, providing players with immediate feedback. Additionally, implemented a feature to generate random chests on the home screen.

Implementation Details

  • UI Manager: The ChestUIManager class manages the user interface for the chest system in a Unity game. It handles the display and update of coins, gems, and other chest-related UI elements, manages interactions such as generating chests, deducting gems, and rewarding players, and communicates with event services to trigger and respond to gameplay events. It also ensures proper initialization of UI elements, updates the reward notification system, and unregisters event listeners when disabled. This class acts as a bridge between the chest system logic and its visual representation to the player
  • Chest Service: The ChestService class manages the backend logic for chest-related functionalities, including generating new chests, handling chest interactions, unlocking chests via timers or gems, processing reward generation based on chest types, and maintaining a queue for sequential chest unlocking, all while interacting with an event service to trigger and manage related events effectively
  • State Machine: The ChestStateMachine class manages state transitions for a chest in a Unity game. It uses a dictionary to store various chest states, associates these states with their corresponding behaviors, and facilitates state changes while ensuring proper entry and exit actions. This class serves as the controller for handling the chest's current state and its transitions dynamically
  • Chest MVC: The MVC for the chest system divided into three distinct components: Model, which defines chest data and its properties; View, which handles the visual representation of chests in the game; and Controller, which manages the interaction logic between the model and the view, including state changes, rewards generation, and player interactions.
