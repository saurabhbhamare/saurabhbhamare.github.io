Bomberman

Take an in-depth look at the detailed code implementation of Bomberman by visiting this Repository link provided for the Project :) GITHUB

Description

  • Bomberman is a thrilling 2D local multiplayer game where players throw bombs at each other in an intense battle of strategy and skill. If caught in the flames of an explosion, the player is eliminated. With fast-paced gameplay and competitive mechanics, Bomberman delivers an exciting and engaging experience for both the players.

Features

  • Blast Radius: Picking up this powerup increases the explosion radius, giving players greater impact on the map. This enhances strategy by allowing smarter moves to outplay opponents.
  • Extra Bomb: Gain an additional bomb immediately upon picking up this powerup.
  • Speed Boost: Temporarily increases your movement speed for quick maneuvers and evasion.

Implementation Details

  • Main Handler: Responsible for managing essential resources and creating necessary services for the game. The resources include Scriptable Object references for players, bombs, explosion data, and tiles. The services include EventService, CharacterService, PowerUpService, and BombService.
  • Character Service: Implemented the CharacterService, a system designed to manage character data, views, and interactions in the game. It connects CharacterSO for character-specific data, CharacterView for visual representation, and CharacterHUD for user interface elements. By initializing and configuring CharacterController instances, the service ensures seamless coordination between character components, creating smooth and engaging gameplay.
  • Bomb Service: Includes functions such as PlaceBomb, ShowExplosion, and Flames, which manage bomb explosions on the map.
  • Event Service: Responsible for Handling Observer Pattern.
  • PowerUp Service: Handles random power-up generation using PowerUpSO and triggers power-up events dynamically during gameplay.
  • MVC Architecture: Used for Character Implementation.
  • Dependency Injection: Implemented dependency injection in the main handler to efficiently manage services and provide the required dependencies.
  • Object Pooling: Implementation of bombs, flames to optimize performance for smoother gameplay and better resource management.
